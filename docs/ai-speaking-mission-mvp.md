# AI Speaking Mission MVP — Historical Note

This Sprint 32 implementation note has been consolidated into the canonical [AI Speaking Mission](AI_SPEAKING_MISSION.md) documentation and [ADR 0004](ADR/0004-ai-speaking-mission.md).

Use those documents for the current contract, security posture, workflow, and known limitations. The stable methodology remains:

```text
Learn → Listen → Repeat → Word Practice → Minimal Pairs → Mixed Practice
→ Sentences → Reading → AI Mission → Quiz
```

The MVP uses extensive in-platform practice, a short external AI mission, approximately 10 + 10 contrast words, 2–3 sentences, one reading, structured paste-back text, no native AI API, and no persisted AI Progress Journal.

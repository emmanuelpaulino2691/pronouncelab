# PronounceLab Development Instructions

## Project

PronounceLab with Emmanuel Paulino is an English pronunciation learning platform.

Slogan: Improve your English every day.

## Technology

- React
- TypeScript
- Vite
- Supabase
- PWA
- GitHub

## Working rules

- Inspect the existing architecture before editing files.
- Reuse existing components, types, services, hooks, and utilities.
- Follow the current folder structure and naming conventions.
- Do not duplicate existing functionality.
- Do not use `any` unless absolutely necessary.
- Do not install dependencies without explicit approval.
- Do not modify environment variables or expose secrets.
- Do not delete working functionality to simplify a task.
- Do not change unrelated files.
- Preserve mobile responsiveness and accessibility.

## Validation

After making changes:

1. Run `npm run build`.
2. Run available tests and lint checks.
3. Fix errors introduced by the changes.
4. Review the final diff.
5. Summarize every modified file.

## Git rules

- Do not commit or push unless explicitly requested.
- Do not switch branches without permission.
- Do not use destructive Git commands.
- Never discard uncommitted work.
- Show `git status` before and after major changes.

## Communication

- Explain the proposed plan before editing.
- Mention which files will likely be affected.
- Report uncertainty instead of guessing.
- Keep explanations concise and practical.
